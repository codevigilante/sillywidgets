Create Kick Ass Websites

Silly Widgets is a static site generator for making client heavy web apps. It uses the concept of widgets, which are bits of reusable HTML to construct complex web sites.

Install

Mac OSX 10.11+

  • Download the Mac archive.
  • Unzip it to a reasonable location.
  • In Terminal, go to the directory where you unzipped the archive.
  • Run chmod +x ./install.sh
  • Then run ./install.sh
  • Enjoy!
Silly Widgets installs to your /usr/local/lib directory and creates a link in /usr/local/bin to the silly executable. This allows you to enjoy silly from wherever you like to develop.

Windows 8.1+

  • Download the Windows archive.
  • Unzip it to a reasonable location.
  • Enjoy!
To make it easier to run the silly command, append the directory where the archive was unzipped to the PATH environment variable.

From Source

  • Download dotnet CLI tools.
  • git clone https://github.com/codevigilante/sillywidgets.git
  • cd into silly directory.
  • dotnet build builds the app, duh.
  • dotnet publish -c Release -r osx.10.11-x64 packages the app for deployment to a specific target, in this case, MacOS 10.11 x64. Other targets can be found in the silly.csproj file.
  • Enjoy!
There's also scripts to publish to specific platforms. Such as publish_mac_10.11.sh. Give one of them a shot if you're feeling lazy.

Use

silly
Shows help and version info, listing all the directives available.
silly new [-location path/to/route/directory] [-name "site-name"]
Initializes a new silly site. Creates a widgets and routes directory in the specified -location, or the current directory if not given. Also creates a default [directory name].json, which is the editable configuration for the site. Setups a default index.html in routes and hello.html in widgets.
silly compile [-location path/to/route/directory]
Checks the silly site for errors, ensuring widgets referenced exist and all assets are accounted for. It also parses the HTML and will throw errors if it's invalid. -location points to the directory where the silly site can be found, otherwise the current directory is used.
silly build [-location path/to/route/directory]
"Shallow" compiles the silly site and starts a development HTTP server. The default port is 7575 and can be changed in [site].json. -location points to the directory where the silly site can be found, otherwise the current directory is used.
silly deploy [-location path/to/route/directory]
Compile and package the silly site for deployment. Creates a deploy directory, resolves all widget references in all the route files, and copies over asset files. This is intended to be uploaded to a static server. -location points to the directory where the silly site can be found, otherwise the current directory is used.

Rules

  • Widgets cannot reference other widgets (yet).
  • Routes can reference widgets only.
  • The directory structure in routes will be the exact same structure that's built for deployment.
